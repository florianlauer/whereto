    - Test: in anonymous mode (user=null), addToWishlist only updates Zustand (no tRPC call)
    - Test: in auth mode (user present), addToWishlist updates Zustand AND calls tRPC client.wishlist.add.mutate
    - Test: in auth mode, removeFromWishlist updates Zustand AND calls tRPC client.wishlist.remove.mutate
    - Test: when user transitions from null to User, fetches server wishlist
    - Test: if server returns items, calls setWishlistItems to replace Zustand
    - Test: if server returns empty, keeps existing local items (no wipe)
    - Test: on mount with existing user (page refresh), fetches server data in background
  </behavior>
  <action>
    Create `src/hooks/useWishlist.ts`:

    1. Read from Zustand: `wishlistItems`, `addLocal` (addToWishlist), `removeLocal` (removeFromWishlist), `setWishlistItems`
    2. Read `user` from authStore
    3. Get tRPC client via `useTRPCClient()` for fire-and-forget mutations (NOT useMutation -- avoids React lifecycle)
    4. Get `queryClient` from `useQueryClient()` and `trpc` from `useTRPC()` for server fetch

    **Auto-fetch on auth transition:**
    - Track previous user via `useRef`
    - When user transitions null -> User: call `queryClient.fetchQuery(trpc.wishlist.list.queryOptions())`, if result has items call `setWishlistItems(items)`, on error log and keep local data

    **Auto-fetch on mount (page refresh):**
    - Separate useEffect with [] deps: if user is already set, fetch server data same way

    **During fetch: no loading flash** -- keep existing Zustand data visible.

    **Mutations:**
    - `addToWishlist(item)`: call `addLocal(item)` first (optimistic), then if `user` exists call `client.wishlist.add.mutate({ poiId: item.poiId, countryCode: item.countryCode, daysMin: item.daysMin }).catch(err => console.error(...))`
    - `removeFromWishlist(poiId)`: call `removeLocal(poiId)` first, then if `user` call `client.wishlist.remove.mutate({ poiId }).catch(...)`

    **Return:** `{ wishlistItems, addToWishlist, removeFromWishlist }`

    API surface per CONTEXT.md: NO clearWishlist (Phase 4), NO helpers (isInWishlist, getCountryItems). Components that need clearWishlist (DestinationPanel, TripSummaryPanel) continue importing it directly from appStore.

    Create tests mocking appStore selectors, authStore, tRPC client, and queryClient.
